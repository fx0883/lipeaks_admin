## 功能概述

### 支持的语言

| 语言代码 | 语言名称 | 标识 | 状态颜色 |
|---------|---------|------|---------|
| `zh-hans` | 简体中文 | 简 | 绿色 |
| `en` | English | EN | 蓝色 |
| `zh-hant` | 繁体中文 | 繁 | 橙色 |
| `ja` | 日本語 | 日 | 红色 |
| `ko` | 한국어 | 한 | 紫色 |
| `fr` | Français | FR | 靛蓝 |

### 可翻译字段

- ✅ **name** - 分类名称
- ✅ **description** - 分类描述
- ✅ **seo_title** - SEO标题
- ✅ **seo_description** - SEO描述

### 共享字段（不翻译）

- **slug** - URL标识符（全局唯一）
- **parent** - 父分类
- **cover_image** - 封面图片
- **sort_order** - 排序值
- **is_active** - 是否激活
- **is_pinned** - 是否置顶
- **tenant** - 所属租户

## 分类列表页面

### 页面布局说明

分类列表页面包含以下列：

| 列名 | 说明 | 示例 |
|------|------|------|
| **Name** | 分类名称（当前语言） | "技术" 或 "Technology" |
| **Slug** | URL标识 | "tech" |
| **Parent** | 父分类 | "技术" 或 null |
| **Is active** | 是否激活 | ✓ 或 ✗ |
| **翻译状态** | 各语言翻译情况 | 🟢简 🔵EN ⚪繁 |
| **文章数** | 关联的文章数量 | 15 |
| **Created at** | 创建时间 | 2025-11-03 12:00:00 |

### 翻译状态标识说明

翻译状态列显示彩色标签：

- **🟢 简** (绿色) = 简体中文已翻译
- **🔵 EN** (蓝色) = 英文已翻译
- **🟠 繁** (橙色) = 繁体中文已翻译
- **🔴 日** (红色) = 日语已翻译
- **🟣 한** (紫色) = 韩语已翻译
- **🔷 FR** (靛蓝) = 法语已翻译
- **⚪ 简** (灰色) = 该语言未翻译

**示例解读**：
- `🟢简 🔵EN ⚪繁 ⚪日 ⚪한 ⚪FR` = 只有中英文翻译
- `🟢简 ⚪EN ⚪繁 ⚪日 ⚪한 ⚪FR` = 只有简体中文翻译
- `🟢简 🔵EN 🟠繁 🔴日 🟣한 🔷FR` = 六种语言都已翻译

### 过滤和搜索

**过滤器**（右侧栏）：
- Is active - 按激活状态过滤
- Parent - 按父分类过滤
- Tenant - 按租户过滤

**搜索框**（顶部）：
- 支持搜索分类名称（所有语言）
- 支持搜索描述内容（所有语言）
- 支持搜索slug

## 创建多语言分类

### 操作步骤

#### 第1步：点击"Add Category"按钮

在分类列表页面右上角，点击 **"ADD CATEGORY +"** 按钮。

#### 第2步：看到语言切换标签

页面顶部会显示语言切换标签：

```
┌────────────────────────────────────────────────────────────┐
│ [简体中文] | English | 繁体中文 | 日本語 | 한국어 | Français │
│  ^^^^^^^^                                                  │
│  当前选中的语言                                             │
└────────────────────────────────────────────────────────────┘
```

#### 第3步：填写共享字段

在 **"基本信息"** 区域填写（只需填一次）：

| 字段 | 是否必填 | 说明 | 示例 |
|------|---------|------|------|
| **Slug** | ✅ 必填 | URL标识，全局唯一 | `programming` |
| **Parent** | ❌ 可选 | 父分类（如有） | 选择"技术"分类 |
| **Cover image** | ❌ 可选 | 封面图片URL | `https://cdn.example.com/prog.jpg` |
| **Tenant** | ✅ 必填 | 所属租户 | 自动选择当前租户 |

在 **"显示设置"** 区域：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| **Sort order** | 排序值，越大越靠前 | 0 |
| **Is active** | 是否激活（前台可见） | ✓ |
| **Is pinned** | 是否置顶 | ✗ |

#### 第4步：填写简体中文翻译

确保 **"简体中文"** 标签被选中（蓝色高亮），在 **"多语言内容"** 区域填写：

| 字段 | 是否必填 | 说明 | 示例 |
|------|---------|------|------|
| **Name** | ✅ 必填 | 分类名称 | `编程` |
| **Description** | ❌ 可选 | 分类描述 | `编程相关的技术文章` |
| **Seo title** | ❌ 可选 | SEO标题 | `编程教程 - 学习编程技术` |
| **Seo description** | ❌ 可选 | SEO描述 | `提供丰富的编程教程和技术文章` |

#### 第5步：切换到English标签

点击页面顶部的 **"English"** 标签，标签会变为蓝色高亮。

#### 第6步：填写英文翻译

在 **"多语言内容"** 区域填写英文版本：

| 字段 | 示例 |
|------|------|
| **Name** | `Programming` |
| **Description** | `Programming related articles` |
| **Seo title** | `Programming Tutorials - Learn Coding` |
| **Seo description** | `Rich programming tutorials and tech articles` |

#### 第7步：切换到繁体中文标签（可选）

如需添加繁体中文翻译，点击 **"繁体中文"** 标签。

#### 第8步：填写繁体中文翻译（可选）

| 字段 | 示例 |
|------|------|
| **Name** | `編程` |
| **Description** | `編程相關的技術文章` |
| **Seo title** | `編程教程 - 學習編程技術` |
| **Seo description** | `提供豐富的編程教程和技術文章` |

#### 第9步：保存

滚动到页面底部，点击 **"SAVE"** 按钮。

✅ 所有语言的翻译会一次性保存！

### 创建成功后

保存成功后会跳转到分类列表页面，你会看到：

- 新创建的分类出现在列表中
- 翻译状态显示已翻译的语言标签（绿色/蓝色/橙色）
- 可以立即在前端API中使用

## 编辑翻译内容

### 快速编辑流程

#### 1. 进入编辑页面

在分类列表中，点击要编辑的分类名称。

#### 2. 查看当前翻译状态

页面会显示 **"翻译状态"** 区域（可能需要展开折叠区域）：

```
┌───────────────────────────────────────────────┐
│ 翻译状态                                       │
├───────────────────────────────────────────────┤
│ 语言      │ 名称     │ 描述     │ SEO标题    │
├───────────────────────────────────────────────┤
│ 简体中文  │ 编程     │ 编程... │ 编程教程...│
│ English   │ Program..│ Program..│ Program... │
│ 繁体中文  │ -        │ -       │ -          │
└───────────────────────────────────────────────┘
```

这个表格显示了所有语言的翻译内容预览。

#### 3. 编辑特定语言

**方法A：切换语言标签编辑**

1. 点击顶部的语言标签（如"English"）
2. 修改 **"多语言内容"** 区域的字段
3. 点击"SAVE"保存

**方法B：快速补充缺失翻译**

1. 查看翻译状态，发现"繁体中文"缺失
2. 点击 **"繁体中文"** 标签
3. 填写name、description等字段
4. 点击"SAVE"

#### 4. 同时编辑多个语言

可以在一次编辑中完成多个语言的更新：

1. 切换到 **"简体中文"** 标签，修改中文内容
2. 切换到 **"English"** 标签，修改英文内容
3. 切换到 **"繁体中文"** 标签，添加繁体内容
4. 点击 **"SAVE"** 一次性保存所有修改

## 批量操作

### 在列表页执行批量操作

#### 激活/停用分类

1. 勾选要操作的分类（左侧复选框）
2. 在顶部"Action"下拉菜单选择操作
3. 点击"Go"按钮

**可用操作**：
- **Activate selected categories** - 批量激活
- **Deactivate selected categories** - 批量停用
- **Delete selected categories** - 批量删除

⚠️ **注意**：批量操作不支持编辑翻译内容。翻译需要逐个编辑。

**原因**：每个分类可能需要不同的翻译内容，无法批量处理。

## 高级功能

### 1. 复制翻译到其他语言

**场景**：已有中文翻译，想快速创建英文版本作为基础。

**步骤**：
1. 编辑分类，切换到 **"简体中文"** 标签
2. 复制name和description内容
3. 切换到 **"English"** 标签
4. 粘贴内容后进行翻译
5. 保存

### 2. 使用翻译工具

**推荐工作流**：

1. 在简体中文标签填写中文内容并保存
2. 重新打开编辑页面
3. 复制中文内容到翻译工具（如Google Translate、DeepL）
4. 将翻译结果粘贴到对应语言标签
5. 人工校对和优化翻译
6. 保存

### 3. 查看翻译历史

Django Admin会记录所有修改历史：

1. 在编辑页面右上角点击 **"HISTORY"** 按钮
2. 查看谁、何时修改了哪些字段
3. 包括翻译字段的修改记录

### 4. 层级分类的翻译

**父子分类的翻译**：

假设有以下结构：
```
技术 (parent)
  └─ Python (child)
  └─ JavaScript (child)
```

**翻译建议**：

**父分类"技术"**：
- 简体中文：技术
- English：Technology
- 繁体中文：技術

**子分类"Python"**：
- 简体中文：Python编程
- English：Python Programming
- 繁体中文：Python編程

**注意**：父分类名称会显示在面包屑导航中，确保翻译准确。

## 常见操作场景

### 场景1：新建完整多语言分类

**目标**：创建一个三种语言都完整的分类。

**步骤**：
```
1. 点击"ADD CATEGORY +"
   ↓
2. 填写共享字段（slug、parent等）
   ↓
3. 在"简体中文"标签填写中文内容
   ↓
4. 切换到"English"标签填写英文内容
   ↓
5. 切换到"繁体中文"标签填写繁体内容
   ↓
6. 点击"SAVE"
   ↓
7. ✅ 完成！三种语言都可用
```

**预期结果**：
- 列表页翻译状态显示：🟢简 🔵EN 🟠繁
- 前端调用任何语言都能获得翻译内容

### 场景2：只创建中文，后续补充英文

**阶段1：快速创建（只填中文）**
```
1. 点击"ADD CATEGORY +"
   ↓
2. 填写slug、tenant等共享字段
   ↓
3. 在"简体中文"标签填写中文name和description
   ↓
4. 点击"SAVE"
   ↓
5. ✅ 分类创建成功，中文用户可用
```

**阶段2：后续补充英文翻译**
```
1. 在列表页点击该分类名称
   ↓
2. 点击"English"标签
   ↓
3. 填写英文name和description
   ↓
4. 点击"SAVE"
   ↓
5. ✅ 英文翻译添加完成
```

**优势**：
- 可以分阶段完成翻译
- 不影响已上线功能
- 灵活安排翻译工作

### 场景3：批量检查翻译完整性

**目标**：找出所有缺少英文翻译的分类。

**步骤**：
```
1. 在分类列表页查看"翻译状态"列
   ↓
2. 找到显示"⚪EN"（灰色EN）的分类
   ↓
3. 记录这些分类的名称或ID
   ↓
4. 逐个编辑补充英文翻译
```

**技巧**：
- 按"翻译状态"列排序（如果支持）
- 使用浏览器搜索功能（Ctrl+F）搜索"⚪EN"

### 场景4：修改已有翻译

**目标**：更新某个分类的英文翻译。

**步骤**：
```
1. 点击分类名称进入编辑
   ↓
2. 点击"English"标签
   ↓
3. 修改name或description字段
   ↓
4. 点击"SAVE"
   ↓
5. ✅ 英文翻译更新完成
```

**提示**：
- 修改不会影响其他语言的翻译
- 可以同时修改多个语言后再保存
- 修改记录会保存在历史中

### 场景5：从slug自动生成初始翻译

**问题**：某些分类只有slug，没有翻译内容。

**快速处理**：

1. 编辑该分类
2. 在 **"简体中文"** 标签：
   - Name填写：根据slug手动输入中文名（如`programming` → `编程`）
   - Description填写：简短描述
3. 在 **"English"** 标签：
   - Name填写：将slug转为可读格式（如`programming` → `Programming`）
   - Description填写：英文描述
4. 保存

## 常见问题

### Q1: 切换语言标签后内容消失了？

**A**: 这是正常的。每个语言标签显示该语言的翻译内容。如果某个语言没有翻译，字段会是空的。

**解决方案**：
- 切换回有内容的语言标签查看
- 或者在当前语言标签填写翻译内容

### Q2: 保存后翻译状态还是灰色？

**A**: 可能的原因：

1. **只填了部分字段**：
   - 必须至少填写name字段才算有翻译
   - 建议name和description都填写

2. **没有切换到正确的语言标签**：
   - 确保在对应语言标签下填写内容
   - 保存前检查当前选中的语言

3. **浏览器缓存**：
   - 刷新页面（F5或Ctrl+R）
   - 清除浏览器缓存

### Q3: 如何删除某个语言的翻译？

**A**: 
1. 编辑分类
2. 切换到要删除翻译的语言标签
3. 清空所有翻译字段（name、description等）
4. 保存

**注意**：如果是默认语言（简体中文），建议保留至少name字段。

### Q4: 翻译状态表格显示乱码？

**A**: 
1. 确保数据库使用utf8mb4字符集
2. 检查浏览器编码设置（应为UTF-8）
3. 联系技术人员检查数据库配置

### Q5: 能否导入导出翻译？

**A**: 当前Admin界面不支持直接导入导出。如需批量操作，可以：

**选项1：使用API**
```bash
# 导出所有分类的翻译
curl -X GET "http://localhost:8000/api/v1/cms/categories/" \
  -H "X-Tenant-ID: 1" > categories_export.json

# 批量更新（需编写脚本）
```

**选项2：直接操作数据库**
```sql
-- 导出翻译数据
SELECT * FROM cms_category_translation 
INTO OUTFILE '/tmp/translations.csv';

-- 导入翻译数据（修改后）
LOAD DATA INFILE '/tmp/translations.csv' 
INTO TABLE cms_category_translation;
```

**选项3：联系技术人员**
请技术人员编写导入导出脚本。

### Q6: 修改slug会影响翻译吗？

**A**: 不会。Slug是共享字段，修改slug不会影响任何语言的翻译内容。

**但要注意**：
- Slug修改后会改变URL
- 前端需要更新相关链接
- 建议谨慎修改slug

### Q7: 如何确保翻译质量？

**建议流程**：

1. **使用专业翻译人员**：
   - 不要完全依赖机器翻译
   - 人工校对很重要

2. **建立翻译规范**：
   - 统一术语翻译（如"分类"始终翻译为"Category"）
   - 保持语言风格一致

3. **同行评审**：
   - 翻译后请另一位翻译人员审核
   - 特别注意专业术语

4. **前端测试**：
   - 在实际页面中查看翻译效果
   - 确保翻译在UI中显示正常

### Q8: 翻译内容过长怎么办？

**A**: 
- **name字段**：最大100字符，如超出需精简
- **description字段**：无长度限制，但建议控制在300字符内
- **seo_title**：最大255字符，建议60字符内（SEO最佳实践）
- **seo_description**：建议150-160字符（搜索结果显示限制）

**如需更长内容**：
- 考虑精简description
- 或联系技术人员调整字段长度限制

## 最佳实践

### ✅ 推荐做法

1. **创建时至少填写简体中文**
   - 确保中文用户能立即使用
   - 其他语言可以后续补充

2. **统一翻译风格**
   - 建立翻译对照表
   - 专业术语保持一致

3. **SEO字段认真填写**
   - seo_title和seo_description影响搜索排名
   - 针对不同语言优化关键词

4. **定期检查翻译完整性**
   - 每周查看翻译状态列
   - 优先补充重要分类的翻译

5. **保持描述简洁明了**
   - description控制在50-100字
   - 突出分类的核心特点

### ❌ 避免的做法

1. **不要留空必填字段**
   - name字段必须填写
   - 空name会导致前端显示异常

2. **不要完全依赖机器翻译**
   - 机器翻译可能不准确
   - 特别是专业术语

3. **不要在翻译中使用HTML标签**
   - description是纯文本字段
   - HTML会被原样显示，不会渲染

4. **不要在不同语言使用相同内容**
   - 如English标签也填中文
   - 失去了多语言的意义

5. **不要忘记保存**
   - 切换标签后修改内容
   - 记得点击SAVE保存

## 故障排查

### 问题：保存后提示错误

**错误类型1**：`Slug already exists`

**原因**：Slug必须全局唯一，已存在相同的slug。

**解决**：
- 修改slug为其他值
- 或检查是否重复创建

**错误类型2**：`This field is required`

**原因**：必填字段未填写。

**解决**：
- 检查slug字段是否填写
- 检查当前语言的name字段是否填写

### 问题：看不到语言切换标签

**可能原因**：

1. **浏览器缓存问题**：
   - 清除浏览器缓存
   - 硬刷新（Ctrl+Shift+R）

2. **django-parler未正确安装**：
   - 联系技术人员检查
   - 确认requirements.txt中有django-parler

3. **Admin配置问题**：
   - 确认CategoryAdmin继承了TranslatableAdmin
   - 检查settings.py中的PARLER配置

### 问题：翻译内容不显示

**检查步骤**：

1. **确认已保存**：
   - 填写内容后是否点击了SAVE

2. **检查语言标签**：
   - 确认在正确的语言标签下填写

3. **查看翻译状态表格**：
   - 展开"翻译状态"区域
   - 确认内容是否保存到数据库

4. **刷新页面**：
   - 保存后刷新页面
   - 重新进入编辑页面查看
